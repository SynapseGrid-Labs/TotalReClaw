# Code of Conduct

## Our standard

This project is maintained as operator-facing infrastructure software. Contributors are expected to keep discussion professional, direct, and respectful.

Examples of acceptable behavior:

- giving specific technical feedback without personal attacks
- disagreeing on implementation details while staying factual
- documenting risks, assumptions, and verification clearly
- helping keep the repo usable for future operators and contributors

Examples of unacceptable behavior:

- harassment or intimidation
- personal insults
- discriminatory language
- deliberately disruptive behavior in issues, pull requests, or discussions

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that violate this code of conduct.

## Reporting

For conduct concerns related to this repository, contact the maintainer through GitHub or the repository security contact path when private handling is needed.
